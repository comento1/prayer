# 구글 시트 연동 (Apps Script)

본 서비스에서 발생한 동작(기도 등록, 함께 기도하기, 댓글, 응답됨 표시, 삭제)을 구글 시트에 기록하려면 아래 순서대로 설정하세요.

## 1. 시트 및 스크립트 설정

1. [Google 시트](https://sheets.google.com)에서 새 스프레드시트를 만듭니다.
2. **확장 프로그램** → **Apps Script** 를 엽니다.
3. `Code.gs` 파일 내용을 이 폴더의 `Code.gs` 내용으로 교체(또는 붙여넣기) 후 저장합니다.
4. **배포** → **새 배포** → **유형 선택**에서 **웹 앱**을 선택합니다.
   - **실행 사용자**: 나
   - **액세스 권한**: 모든 사용자 (또는 조직 내 사용자)
5. **배포**를 누르고 생성된 **웹 앱 URL**을 복사합니다.

## 2. 서버 환경 변수

프로젝트 루트의 `.env` 파일에 다음을 추가합니다.

```
GOOGLE_SHEETS_WEBHOOK_URL=https://script.google.com/macros/s/여기에_복사한_URL/exec
```

서버를 재시작하면 이후부터 기도 등록·함께 기도·댓글·응답됨·삭제가 시트 **중보기도_기록** 시트에 자동으로 한 줄씩 추가됩니다.

## 3. 시트 컬럼

**중보기도_기록** (이벤트 로그):

| 시각 | 동작 | 사용자ID | 닉네임 | 기도ID | 상세1 | 상세2 |
|------|------|----------|--------|--------|-------|-------|
| ISO 시각 | prayer_created / pray / comment / answered / prayer_deleted | | | | 내용 요약 | (groupId 등) |

**사용자** (로그인/가입용, `action: 'auth'` 시 자동 생성):

| 닉네임 | PIN | 생성일시 |
|--------|-----|----------|
| 사용자가 입력한 이름 | PIN | ISO 시각 |

- 최초 입력 시: 해당 행이 시트에 추가되고 로그인 처리됩니다.
- 이후 같은 닉네임으로 접속 시: 시트에 기록된 PIN과 일치하면 로그인, 아니면 "PIN이 일치하지 않습니다." 오류를 반환합니다.

## 4. Vercel 등 정적 배포에서 로그인만 쓰는 경우

프론트에서 API 서버 없이 로그인만 쓰려면, Vercel 환경 변수에 **`VITE_GOOGLE_SHEETS_WEBHOOK_URL`** 을 위 웹 앱 URL과 동일하게 설정하세요. 그러면 로그인/가입이 구글 시트 '사용자' 시트와 연동됩니다.

`GOOGLE_SHEETS_WEBHOOK_URL`을 설정하지 않으면 시트 기록은 하지 않고, API 동작은 그대로 유지됩니다.
